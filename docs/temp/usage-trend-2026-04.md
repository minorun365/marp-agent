## 7. データ取得上の注記

### 取得元・クエリ

- **Cognito 新規ユーザー**: `aws cognito-idp list-users` で全件ページング取得し、`UserCreateDate` を JST 変換して月別カウント
- **AgentCore セッション**: CloudWatch Logs Insights で OTEL ログから `session.id` を抽出し `count_distinct` を月別集計（各ロググループに対し個別クエリ）
- **Bedrock コスト**: Cost Explorer API `GetCostAndUsage` / `MONTHLY` / `UnblendedCost` / `RECORD_TYPE=Usage`（クレジット適用前）

### 制約・誤差

- **sandbox アカウントは marp-agent 専有ではない**: voice_agent / nova_sonic など他 Runtime も同居。`main + dev` のコストは **やや上振れ気味**。一方 `kag-sandbox` は marp-agent ほぼ専有のため信頼度高。
- **ロググループ retentionDays**: 全ロググループで `null`（失効なし）。保持期間による欠損はなし。
- **月またぎセッション**: 通常は短時間で閉じるため誤差は小さいが、厳密には両月にカウントされる可能性あり。
- **kag プール**: 2/10 にアカウント移行。2 月の kag セッション 117 は旧（61）+ 新（56）の合算値。
- **セッション ID の抽出**: `session.id` 属性を正規表現 parse。Strands Agents 側でのセッション定義に依存。

### Cost Explorer 再現コマンド（参考）

```bash
aws ce get-cost-and-usage \
  --time-period Start=2026-01-01,End=2026-05-01 \
  --granularity MONTHLY \
  --metrics UnblendedCost \
  --filter '{"And":[{"Dimensions":{"Key":"SERVICE","Values":["Amazon Bedrock"]}},{"Dimensions":{"Key":"RECORD_TYPE","Values":["Usage"]}}]}' \
  --profile sandbox --region us-east-1
```
